# Product Backlog: VARTA (Distributed Resilience Orchestrator)

**Product Owner:** Викладач (роль PO)
**Пріоритезація:** Critical / High / Medium / Low

---

## Опис продукту

**VARTA** — децентралізована мережа управління критичними ресурсами (енергія, вода, зв'язок) для міських громад в умовах повної або часткової ізоляції інфраструктури.

### Чому це важливо

- **Технічна складність:** Offline-first архітектура, синхронізація через Mesh-мережі, цілісність даних без центрального сервера.
- **Соціальна вага:** Пріоритезація ресурсів у критичних умовах (кому дати залишок заряд спільної батареї: серверу зв'язку чи медичному обладнанню?).
- **Відсутність аналогів:** Це не карта і не чат — це динамічний балансувальник ресурсів з елементами теорії ігор.

### Контекст використання

Мешканці міської громади, відрізаної від центральної інфраструктури. У кожного — смартфон або мікроконтролер (Raspberry Pi / ESP32), підключений до локальної Mesh-мережі (Bluetooth LE / Wi-Fi Direct). Центральний сервер **відсутній** або **недоступний**. Рішення мають прийматися **локально** та **автоматично**.

## User Stories (Пріоритезований Backlog)

> 💡 **Кросфункціональні Сквади та їхній фокус:**
> Згідно з принципами Agile та для уникнення "функціональних колодязів" (Закон Конвея з Лекції 2), **кожен сквад є повністю кросфункціональним** (має у складі Dev, QA, Ops тощо) і відповідає за реалізацію фічі "під ключ" (End-to-End). Проте, для зручності вибору завдань на воркшопі, сквади мають свій *тематичний фокус*:
> * **Сквад Alpha** — Фокус: UX / Flow. Команда береться за E2E-фічі, де головним викликом є взаємодія з користувачем.
> * **Сквад Beta** — Фокус: Domain Logic. Команда реалізує фічі, де ядром є складна алгоритмічна розробка та бізнес-правила.
> * **Сквад Gamma** — Фокус: Performance / Ops. Команда "тягне" фічі з високими вимогами до надійності та швидкодії.
> *(Важливо: Gamma теж робить UI для своєї фічі, а Alpha теж пише тести і налаштовує логіку. Жоден сквад не є 'вузьким відділом').*

### EP-01: Mesh Identity & Trust

| ID | Name | User Story | Рекомендовані Сквади | Пріоритет |
| :--- | :--- | :--- | :--- | :--- |
| US-01 | Offline Registration | Як мешканець, хочу зареєструвати свій пристрій у локальній мережі **без інтернету**, щоб стати учасником громади VARTA. | Alpha, Beta | **Critical** |
| US-02 | Face-to-Face Trust | Як мешканець, хочу підтвердити особу іншого учасника через механізм «поруч-факту» (QR-код при особистій зустрічі), щоб побудувати Web of Trust без центрального сервера. | Alpha, Beta | **Critical** |
| US-03 | Revoke Trust | Як адміністратор вузла, хочу відкликати довіру до скомпрометованого пристрою, щоб захистити мережу від зловмисника. | Beta, Gamma | **High** |
| US-04 | ZKP Role Proof | Як мешканець, хочу мати можливість довести свою роль (медик, інженер) без розкриття персональних даних (ZKP), щоб отримати відповідний пріоритет доступу до ресурсів. | Gamma | **Medium** |

## Обмеження та Constraints

| Constraint | Опис |
| :--- | :--- |
| **Offline-first** | Система ПОВИННА працювати без інтернет-з'єднання. Центральний сервер — опція, а не вимога. |
| **Mesh Transport** | Передача даних через Bluetooth LE або Wi-Fi Direct. Пропускна здатність ≤ 250 Kbps. |
| **Hardware** | Мінімальний пристрій: смартфон Android 8+ або Raspberry Pi 3B+ з 1 GB RAM. |
| **Енергоефективність** | Фонові процеси не повинні розряджати батарею смартфона швидше, ніж на 5% на годину. |
| **Безпека** | Всі дані шифруються end-to-end. Жодний вузол не має доступу до даних іншого вузла без явного дозволу. |
| **Регуляторні** | Система не замінює екстрені служби (112). Рішення диспетчера — рекомендаційні, не директивні. |
