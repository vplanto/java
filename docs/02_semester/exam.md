# Екзаменаційний пул питань (Java Engineering & System Design)

### Тема 1: Engineering Mindset та Основи SDLC
1. У чому фундаментальна різниця між "Кодером" (Coder) та "Інженером" (Engineer)?
2. Поясніть концепцію "Code is a Liability" (Код — це пасив).
3. Що таке "AI Ownership" у контексті Vibe Coding Protocol і чому розробник несе відповідальність за згенерований код?.
4. Чому методологія Waterfall вважається "небезпечною" для стартапів, але обов'язковою для сфер із високою ціною помилки (медицина, космос)?.
5. У чому принципова різниця між Scrum і Kanban? Наведіть приклад, де ви б обрали кожен з цих фреймворків.

### Тема 2: Архітектура Java та JVM
1. Поясніть принцип "Write once, run anywhere". Яку роль відіграють компілятор, байт-код, інтерпретатор та JIT-компілятор?.
2. У чому полягає ключова відмінність між Купою (Heap) та Стеком (Stack) в JVM і як їхній життєвий цикл пов'язаний з потоками?.
3. Алгоритм збирання сміття "Reference Counting" має фундаментальну проблему при управлінні пам'яттю. Назвіть цю проблему та алгоритм, який її вирішує.
4. Що таке Пул констант (Constant Pool) у файлі `.class` і яку роль він відіграє при виконанні коду?.

### Тема 3: Інженерія Вимог (Requirements)
1. Чим User Story відрізняється від технічного завдання (ТЗ)?.
2. Що таке Acceptance Criteria, навіщо вони потрібні та як вони пов'язані з форматом Given/When/Then?.
3. Поясніть принцип «вертикального порізання» User Stories. Чому горизонтальне поділення по технічних шарах є антипатерном?.
4. Що таке INVEST і як критерій «Valuable» відрізняє справжню User Story від простої технічної задачі?.
5. Наведіть приклад «невидимої» (BAU) вимоги і поясніть, чому замовник зазвичай не говорить про неї сам.

### Тема 4: Нефункціональні Вимоги (NFRs)
1. Чому «система має бути швидкою» — це не NFR? Як сформулювати цю вимогу правильно для архітектора?.
2. Поясніть різницю між Vertical і Horizontal Scaling. Коли горизонтальне масштабування стає неможливим без зміни архітектури додатка?.
3. Що таке SLA і що таке SLO? Як пов'язані ці поняття у контексті Availability?.
4. Що таке Architectural Decision Record (ADR) і навіщо фіксувати «чому ні» (відкинуті альтернативи) поряд із «чому так»?.

### Тема 5: ООП, SOLID та Патерни проєктування
1. Чому поля класу прийнято оголошувати як `private`, а доступ до них надавати через `public` геттери та сеттери (концепція Інкапсуляції)?.
2. Як використання наслідування або композиції (патерн Decorator) дозволяє дотримуватися Принципу відкритості/закритості (OCP)?.
3. Поясніть своїми словами Принцип інверсії залежностей (DIP) та як він реалізується через Dependency Injection (DI).
4. Який патерн дозволяє динамічно змінювати алгоритм роботи об'єкта залежно від ситуації (наприклад, різні розрахунки податку)?.
5. Чому патерн Singleton часто називають антипатерном, особливо у контексті мікросервісів та тестування?.

### Тема 6: Колекції в Java (JCF) та Generics
1. Назвіть три основні інтерфейси, що наслідуються від `Collection`. Яка їхня ключова відмінність щодо збереження порядку елементів та обробки дублікатів?.
2. Чому інтерфейс `Map` не є частиною ієрархії `Collection`?.
3. Чому операція додавання елемента в середину `ArrayList` має складність O(n), а в `LinkedList` — O(1)?.
4. Ви хочете зберігати унікальні об'єкти, але вам важливо обходити їх у тому порядку, в якому ви їх додали. Яку реалізацію `Set` ви оберете і чому?.
5. У чому полягає ключова відмінність між Generics у Java (Type Erasure) та шаблонами (templates) у C++ (Генерація коду)?.

### Тема 7: Багатопотоковість (Concurrency)
1. У чому полягає "стан гонитви" (race condition) і чому операція `count++` є небезпечною без синхронізації?.
2. Яку гарантію надає ключове слово `synchronized` (монітор) і якою "ціною" для продуктивності інших потоків це досягається?.
3. Назвіть три причини, чому створення `new Thread()` для кожного запиту на сервері — це шлях до катастрофи..
4. Що робить метод `executor.shutdown()` для пулу потоків і чому його виклик є обов'язковим для коректного завершення програми?.
5. У чому ключова відмінність між інтерфейсами `Runnable` та `Callable` при роботі з потоками?.

### Тема 8: Винятки (Exceptions), Логування та Troubleshooting
1. У чому полягає ключова різниця між `Error` та `Exception` в Java? Наведіть приклади.
2. Поясніть різницю між Checked та Unchecked exceptions. Чому компілятор змушує обробляти одні, але не змушує інші?.
3. Поясніть "головний конфлікт" логування між розробником і продакшеном. Як рівні логування (Levels) допомагають його вирішити?.
4. Чому використання конкатенації `LOG.debug("Data: " + data)` є неефективним, і як плейсхолдери `SLF4J` вирішують цю проблему?.
5. Що таке MDC (Mapped Diagnostic Context) у логуванні і яку проблему він вирішує у багатопотокових веб-сервісах?.

### Тема 9: Бази даних та JDBC
1. Що таке Теорема CAP? Які два компроміси найчастіше обирають NoSQL та розподілені бази даних при виникненні мережевого поділу?.
2. У чому полягають основні відмінності між реляційними базами даних (SQL) та NoSQL рішеннями? Наведіть приклади типів NoSQL баз.
3. Чому використання `PreparedStatement` у JDBC є обов'язковим з точки зору безпеки порівняно зі звичайним `Statement`?.
4. Назвіть 5 основних кроків життєвого циклу JDBC-запиту в Java.

### Тема 10: API Design
1. Поясніть різницю між HTTP-методами `PUT` і `PATCH`. Чому використання `PUT` для часткового оновлення є архітектурною помилкою?.
2. Поясніть різницю між статус-кодами `401 Unauthorized` і `403 Forbidden`. Наведіть конкретний сценарій для кожного.
3. Що таке breaking change в публічному API? Наведіть по два приклади breaking і non-breaking змін у структурі JSON-відповіді.

5. Яку архітектурну проблему вирішує Contract-First підхід (OpenAPI) у розподіленій команді під керівництвом Integration Lead?.

### Тема 11: QA Стратегія та Тестування
