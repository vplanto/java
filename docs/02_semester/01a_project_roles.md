# Лекція 1a: Project Roles and Responsibilities

**Аудиторія:** 2-й курс (Junior Strong)
**Ціль:** Ознайомитися з розширеним переліком ролей в Enterprise Delivery.

> **English version:** [English](en/01a_project_roles.md) | **Незнайоме слово?** → [Глосарій курсу](glossary.md)

---

На реально великих проєктах (аутсорс або масштабні інтеграції) структура управління стає складнішою. Ось ключові ролі, з якими ви будете стикатися в Enterprise Delivery:

| Роль | Ключова відповідальність (Key responsibilities) |
| :--- | :--- |
| **Account Manager** | Наглядає за клієнтом з точки зору продажів (Sales perspective): успішність поточних проєктів, пошук нових можливостей для продажу. |
| **Program Director** | Координатор та адміністратор усіх аспектів клієнтської програми. Керує програмою з управлінської точки зору, координує проєктні активності, ресурси та віхи (milestones) між багатьма паралельними проєктами одного замовника. |
| **Engineering Sponsor** | Ескалація та координація всіх питань виключно з інженерної (Software Engineering) точки зору. |
| **Project Manager On-Site** | Відповідає за загальне управління проєктом "на місці" (ближче до клієнта). Головна контактна особа (Principal point of contact) для клієнта. Працює з PM клієнта для узгодження планів. Уточнює функціональні вимоги, фіксує прийняті рішення та обмеження систем. |
| **Project Manager Off-Site** | Знаходиться разом із командою розробки в Delivery-центрі. Забезпечує технічні ресурси для проєкту, працює в парі з On-Site PM, переконуючись, що результати відповідають очікуванням. |
| **Solution Architect** | Проєктує загальну архітектуру (Solution Architecture), моніторить та керує імплементацією. Відповідає за наскрізний (End-to-End) архітектурний дизайн. |
| **Cloud Architect** | Специфічний фокус на E2E Cloud Solution. Проєктує хмарну екосистему та архітектуру в хмарі. |
| **Technical Manager** | Відповідає за технічний дизайн рішення, налаштування продукту (configuration & customization). Головний технічний контакт для замовника, перевіряє що прототипи та дизайн-специфікації відповідають очікуванням. |
| **BA Lead** (Business Analyst Lead) | Відповідає за визначення вимог разом із Stream Leads та експертами. Гарантує наскрізне відстеження вимог (Requirements traceability) протягом усіх фаз проєкту. |
| **Stream / Dev / Integration Lead** | Технічний лідер конкретного напрямку роботи (Stream) — наприклад, розробка, інтеграція чи міграція даних. Координує роботу розробників у своєму напрямку. |
| **QA Manager** | Координує роботу команди QA і несе відповідальність за якість релізу (code drop deliverables). |
| **SME** (Subject Matter Expert) | Предметний експерт. Надає команді глибоку експертизу в специфічній бізнес-області, що допомагає приймати правильні рішення під час розробки (Solution Delivery). |
